# Краткое руководство по Flux Auto-Deleter

Это руководство поможет вам быстро понять назначение приложения, настроить его и запустить в работу.

## 1. Назначение приложения

**Flux Auto-Deleter** — это настольное приложение для мониторинга ваших узлов (нод) Flux в локальной сети. Его главная задача — автоматически удалять указанные вами приложения с узла после того, как вы войдете в его веб-интерфейс.

Приложение отображает веб-интерфейс каждого найденного узла в отдельной вкладке и показывает единый лог всех автоматических действий.

## 2. Установка

Перед установкой убедитесь, что на вашем компьютере установлен **Node.js**.

Откройте терминал (командную строку) в папке с проектом и выполните одну команду:
```bash
npm install
```
Эта команда скачает и установит все необходимые компоненты.

## 3. Настройка

Вся настройка выполняется в одном файле: `settings.ini`. Откройте его в любом текстовом редакторе. Ниже описаны только самые важные параметры, которые нужны для начала работы.

#### `ScanIPs`
Это самая важная настройка. Здесь вы должны перечислить IP-адреса ваших узлов Flux через запятую.

**Важно:** Не используйте пробелы между адресами.

*Пример:*
```ini
ScanIPs = 192.168.1.10,192.168.1.15,192.168.1.20
```

#### `TargetAppPrefixes`
Здесь нужно перечислить имена (или начала имен) тех приложений, которые вы хотите автоматически удалять. Имена также перечисляются через запятую.

Приложение будет удалять любую программу, чье имя *начинается* с указанных здесь префиксов. Регистр (большие или маленькие буквы) не имеет значения.

*Пример:*
Если указать `TargetAppPrefixes = Folding,Kaspa`, приложение будет удалять программы с именами `FoldingAtHome`, `folding-app`, `TestApp`, `kaspa-mainnet` и так далее.

```ini
TargetAppPrefixes = StuckContainer,MaliciousApp,TestApp
```

#### `AutomationIntervalSeconds`
Это интервал в секундах, с которым приложение будет проверять и удалять целевые приложения после вашего входа на узел.

Например, значение `300` означает, что проверка будет выполняться каждые 5 минут. Не рекомендуется ставить слишком низкое значение, чтобы избежать чрезмерной нагрузки на сеть.

```ini
AutomationIntervalSeconds = 300
```

## 4. Запуск

После установки и настройки, запустите приложение командой:
```bash
npm start
```

## 5. Как это работает

1.  При запуске приложение сканирует IP-адреса, которые вы указали в `settings.ini`.
2.  Для каждого найденного узла открывается отдельная вкладка с его стандартным веб-интерфейсом.
3.  Выполните вход в свой узел как обычно, используя логин и пароль в соответствующей вкладке.
4.  Как только вы войдете, приложение обнаружит это и начнет в фоновом режиме удалять приложения, указанные в `TargetAppPrefixes`, с интервалом, заданным в `AutomationIntervalSeconds`.
5.  Все действия (обнаружение узлов, удаление приложений, ошибки) отображаются в реальном времени на панели **Logs** в нижней части окна.
